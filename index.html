How I work

Every problem gets sorted into the same 3 buckets. When a project is stuck, one of these is usually the reason.

01

Business value

What decision does this change, and what is that worth? If I can't answer in a sentence, it's a demo. Demos are fun to build and easy to kill at budget time.

02

Technical feasibility

Can we build it with the stack and the data we actually have? I prototype before I promise. The tech is usually the easy part, which still surprises people.

03

Organizational dynamics

Who has to change how they work for this to land? This is where AI projects die. Someone has to sit with the team that distrusts the new tool, and I'd rather it be the person who built it.

AI Investment Map

The business problem

A board asks 3 questions about AI: how much have we spent, is it working, and where do we need to decide? With about 130 initiatives tracked by hand, nobody can answer from one screen.

Tech stack

  • React 19
  • Vite 7
  • Tailwind CSS 4
  • Hand-rolled squarified treemap
  • Vercel

What I learned

The treemap is hand-rolled, no chart library. That one decision made everything else possible: nested business-unit regions, drill-down transitions, semantic zoom, and a map that rescales smoothly as the timeline moves.

Brief Me

Deep Research Harness

The business problem

Standing research questions need re-answering every quarter, and one-off reports start from zero each time. This pipeline reruns, diffs against prior runs, and keeps its bibliography deduped.

Tech stack

  • Claude Code
  • Slash commands
  • Markdown
  • No database, no runtime

What I learned

Subagents never return findings inline. Full findings go to disk and the orchestrator only holds summaries, so the context budget survives no matter how much the subagents dig up. Every claim carries a URL and a date or it's out.

Chief of Staff OS

The business problem

A talking-points cycle eats 1 to 2 weeks of a chief-of-staff team's time, and generic LLM output fails twice: it sounds like nobody and it invents numbers.

Tech stack

  • System prompts
  • Voice guides
  • Content-block library
  • Numbers firewall
  • Holdout validation

What I learned

The tech was the easy part. The hard part was deciding what the model is allowed to do, and proving the system works before asking anyone to trust it. The firewall prevents fabrication instead of catching it after.

Agentic PMO Playbook

The business problem

Monthly portfolio reporting burns weeks: the PMO chases status, half the replies come late in 6 formats, and the deck ships with stale numbers. Here the report becomes a byproduct of small notes logged in Teams.

Tech stack

  • Copilot Studio
  • Dataverse, 6 tables
  • Power Automate, 10 flows
  • React + Fluent UI
  • Teams

What I learned

Adoption came from meeting people in Teams: no portal, no new login, friction measured in seconds. And the sign-off design carried more weight than any feature. A human owns the AI-drafted words before their name attaches.

IR Memo Agent

The business problem

IR teams hand-build a briefing memo before every analyst meeting. A 15 to 20 analyst roster costs hundreds of hours a year in retrieval and formatting.

Tech stack

  • FastAPI
  • Claude API
  • yfinance
  • SEC EDGAR
  • SSE streaming
  • React 19

What I learned

Every section of the memo comes from a different source, and each source can fail. So it's an agent pipeline with explicit fallbacks, confidence badges, and a final quality check. Gaps become labeled placeholders, never invented facts.

Spelling Bee

The business problem

My kid wanted spelling practice to feel like a Pokemon game. Voice-first, no typing: the app reads the word, the kid spells it out loud, and every letter gets checked.

Tech stack

  • React 19 + TypeScript
  • OpenAI TTS + Whisper
  • Vercel serverless
  • PWA, offline-ready

What I learned

The UI knows nothing about storage or API calls, keys live server-side only, and progress sits in localStorage behind an interface a real backend could replace. Small and finished counts too.
